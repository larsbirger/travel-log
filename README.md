# App template

AGPL web app template

> This template is designed for an app that needs to comply with AGPLv3.0.  
>
> > That being said, i will not sue you, nor pursue legal action
> > if you just take the template, refurbish it for something else,
> > and just run it on a different license so it
> > might be closer to an open license.
>
> Most, if not all of this template, is autogenerated using AI.  
> I will not hide the use of AI when it is used,
> but i try to maintain quality where AI als short
> and learn from AI where i all short

## Requirements

- A machine (virtual or physical or containerized):
  - Running Linux
    (I only test debian on lxc and physical machine, but you do you).
  
  - Have git installed.
  
  - Preferably access to internet
    (not necessarily required if you want to manually install
    NGINX, Podman, and your backend offline).
  
  - Preferably access to any kind of network
    (not necessarily required if you only want to use
    the web app on the machine running it).

## General folder structure

```plaintext
my-agpl-project/
├── .gitignore
├── LICENSE                 # The AGPL-3.0 License file
├── README.md               # Quick setup instructions for users, and additional trivia
├── setup.sh                # The single-command installation script
│
├── backend/                # Your core API service
│   ├── src/                # Rust (.rs) or TS (.ts) source files
│   ├── Cargo.toml / package.json
│   └── Containerfile       # Builds the backend into an image
│
├── frontend-web/           # The simple web form page
│   └── index.html          # Served directly by Nginx
│
└── deploy/                 # Everything needed to run on the server
    ├── nginx.conf          # Nginx reverse-proxy rules
    ├── app.container       # Quadlet: Runs your backend container
    ├── nginx.container     # Quadlet: Runs your Nginx container
    ├── app.network         # Quadlet: Internal network linking them
    └── app.env.tmpl        # Template for user secrets
```

If you make changes to this template folder structure, be it changes in folders, files, or semantic changes in usage of folders and files, then please update this "chart".

## In general how to use this template

- Put your backend app in the `/backend` folder, and route nginx to it.
- Commit everything to your personal Git repository.
- On your clean production machine, download your project repository and initialize:

    ```bash
    git clone <your-repo-url>
    cd <your-repo-folder>
    sudo bash setup.sh
    ```
